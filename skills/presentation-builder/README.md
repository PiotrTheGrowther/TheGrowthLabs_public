# Presentation Builder

A Claude skill that turns written content (YouTube transcripts, PDFs, markdown/docx/txt docs, Notion pages) into a **single self-contained vertical scroll-snap HTML site** that works as a presentation.

No generic AI aesthetics. No purple gradients on white. No Inter on Roboto. Pick a bold aesthetic direction, commit to it, ship.

## What you get

- **Single .html file** with CSS and JS inlined - one file to share, attach, or host
- **Vertical scroll-snap** - full-viewport sections, smooth keyboard nav (PgDn / ArrowDown / Space)
- **11 section types**: hook · metric · before-after · quote · steps · timeline · logo-grid · comparison · terminal · workspace · cta
- **5 aesthetic directions**: editorial · brutalist · retro-futuristic · luxury · industrial (one per deck, committed precisely)
- **Brand layer is opt-in**: drop logos in `assets/`, name the brand, palette + marks get applied
- **Self-verifies** before handing back: em-dash check, font check, AI-vocab check, asset 404 check

## Install

### Claude Cowork (recommended for non-devs)

1. Download `presentation-builder.zip` from the parent folder ([direct link](../presentation-builder.zip))
2. Open Claude Cowork
3. Settings → Capabilities → Skills → Add Skill
4. Drag the zip into the upload area
5. Done - the skill is now available in every new Cowork task

### Claude Code (CLI)

```bash
# Clone or download this folder, then:
cp -r presentation-builder ~/.claude/skills/
```

The skill auto-loads on next session.

## Use it

Open a folder containing your source content (a transcript, brief, doc, or PDF). Prompt:

> Use the presentation-builder skill to turn this transcript into a vertical site.

Claude reads the skill, finds the file, outlines 8-14 sections, picks an aesthetic, generates the HTML, and writes it to `./presentations/<slug>.html`. Total time: about 3 minutes.

### Useful follow-up prompts

| You say | Claude does |
|---|---|
| *"swap the aesthetic to brutalist"* | Re-renders the same outline with the brutalist design system |
| *"now apply our brand"* (with `assets/logos/` present) | Sources brand colors from the logo, embeds the mark, updates CSS variables |
| *"rewrite section 3 - too wordy"* | Tightens copy in that section only, keeps the layout |
| *"target 8 sections, denser"* | Re-outlines shorter, more content per section |
| *"add a comparison section between A and B"* | Inserts a new section, regenerates the file |

## Inputs supported

| Input | How |
|---|---|
| YouTube transcript (txt/md) | Chapter markers auto-detected; falls back to topic-shift clustering |
| Markdown / .txt | Heading structure becomes the section skeleton |
| PDF | Text extracted, sections reconstructed from heading patterns |
| .docx | Converted via pandoc when available; else ask to export as md |
| Notion page (URL) | Fetched via Notion MCP, heading blocks become sections |

## Hard rules baked in

- No em-dashes (`—` is replaced with `-` everywhere)
- No Inter, Roboto, Arial, system-ui, Helvetica as primary fonts
- No purple-to-pink gradients on white
- No banned AI vocab: *delve, leverage, unlock, navigate the complexities, evolving landscape, stands as a testament, seamless, vibrant, robust solution*
- File size budget: under 200 KB
- All CSS and JS inlined - one file, no external dependencies (except Google Fonts)

## Tweak it

Every rule lives in a markdown file you can edit. Want a different aesthetic? Add one to `references/design-system.md`. Want a new section type? Drop a `templates/section-mything.html` and document it in `references/slide-types.md`. The skill picks it up immediately - no rebuild needed.
